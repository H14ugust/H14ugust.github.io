Java多线程

1.多线程优点

  • 线程占用系统资源比进程少
  • 线程共享内存，而进程不能

2.Java实现多线程的方式

  • 继承Thread类
  • 实现Runnable接口
  • 实现Callable接口

2.1继承Thread类

两个独立的Thread对象，分别实现计数。

缺陷：1.受限于Java的单继承特性

​ 2. run方法不返回值

2.2实现Runnable接口

实现Runnable接口后，需要使用Thread类来启动

Thread类，有一个构造方法，参数是Runnable对象

两个Thread对象共享一个Runnable对象,两个Thread共同完成计数。两个Runnable对象则可实现两个Thread独立工作。

缺陷：1. 线程不安全，需要加入同步操作（互斥锁），保证每次只有一个线程占用for循环内的操作

​ 2. run方法不返回值

2.3 实现Callable接口

Callable对象需要和Future对象（异步）结合使用。使用Future对象的get方法获取返回值

call方法可抛出异常，run方法不能

Callable是泛型接口

3.Thread的run和start

  • start() : 它的作用是启动一个新线程，新线程会执行相应的run()方法。start()不能被重复调用。
  • run() : run()就和普通的成员方法一样，可以被重复调用。单独调用run()的话，会在当前线程中执行run()，而并不会启动新线程！

4.synchronized

  • 重量级锁
  • 可重入

synchronized可以用在方法上也可以使用在代码块中，其中方法是实例方法和静态方法分别锁的是该类的实例对象和该类的对象。而使用在代码块中也可以分为三种，具体的可以看上面的表格。这里的需要注意的是：如果锁的是类对象的话，尽管new多个实例对象，但他们仍然是属于同一个类依然会被锁住，即线程之间保证同步关系
